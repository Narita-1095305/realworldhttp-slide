HTTPステータスコードとフィールド一覧

本書で解説したHTTPステータスコードとヘッダーフィールドを一覧形式で網羅的に解説します。

100番台: 情報

ステータス 用途
100 Continue コンテンツ送信前にサーバーの許可を得た。
101 Switching Protocols プロトコル変更をサーバーが受理した。
103 Early Hints 本体のレスポンスの前に、先行して読み込むべきリソースを通知する。

200番台: 成功

ステータス用途
200 OK 正常終了。最も一般的なレスポンス。
201 Created リソースの作成に成功（主にPOSTへの応答）。
202 Accepted リクエストは受理されたが、処理は完了していない。
203 Non-Authoritative Information プロキシが中継したが、そのプロキシは信頼できない。
204 No Content 成功したが、返すコンテンツはない（例: DELETE成功時）。
205 Reset Content クライアントの画面リセットを要求。
206 Partial Content 範囲指定ダウンロードで、一部のコンテンツを返す。

300番台: リダイレクト

ステータス用途
300 Multiple Choices リソースの選択肢が複数ある。
301 Moved Permanently 恒久的な移転。メソッド変更が可能。
302 Found 一時的な移転。メソッド変更が可能。
303 See Other 他ページへのリダイレクト。キャッシュされない。
304 Not Modified キャッシュが有効であることを示す。
305 Use Proxy 指定されたプロキシを使用する必要がある。（非推奨）
307 Temporary Redirect 一時的な移転。メソッド変更には許可が必要。
308 Permanent Redirect 恒久的な移転。メソッド変更には許可が必要。

400番台: クライアントエラー (1/2)

ステータス用途
400 Bad Requestリクエストに問題がある。
401 Unauthorized認証が必要。
402 Payment Required支払いが必要。（予約済み）
403 Forbiddenアクセスが禁止されている。
404 Not Foundリソースが見つからない。
405 Method Not Allowed許可されていないメソッド。
406 Not Acceptable受理できない形式。
407 Proxy Authentication Requiredプロキシの認証が必要。
408 Request Timeoutリクエストがタイムアウトした。
409 Conflictリソースの競合。
410 Goneリソースは恒久的に存在しない。

400番台: クライアントエラー (2/2)

ステータス用途
411 Length RequiredContent-Lengthが必要。
412 Precondition Failed事前条件に失敗した。
413 Payload Too Largeペイロードが大きすぎる。
414 URI Too LongURIが長すぎる。
415 Unsupported Media Typeサポートされていないメディアタイプ。
416 Range Not Satisfiable範囲指定が無効。
417 Expectation FailedExpectヘッダーの期待に応えられない。
421 Misdirected Request誤ったリクエスト。
425 Too EarlyTLSハンドシェイク中のリプレイ攻撃の可能性。
426 Upgrade Requiredプロトコルのアップグレードが必要。
428 Precondition Required事前条件が必要。
429 Too Many Requestsリクエストが多すぎる。
431 Request Header Fields Too Largeリクエストヘッダーが大きすぎる。
451 Unavailable For Legal Reasons法的理由により利用不可。

500番台: サーバーエラー

ステータス用途
500 Internal Server Errorサーバー内部エラー。
501 Not Implemented未実装の機能。
502 Bad Gateway不正なゲートウェイ。
503 Service Unavailableサービス利用不可（メンテナンス等）。
504 Gateway Timeoutゲートウェイがタイムアウト。

フィールド一覧について

この後のスライドでは、本書で解説したHTTPヘッダーフィールドを機能ごとに分類して一覧で紹介します。

完全なリストはIANAの公式サイトで確認できます。

基本のフィールド

フィールド名説明
Host送信先のホスト名。バーチャルホストに必須。
Refererページ遷移元のURL。
Dateレスポンスを返した日時。
Locationリダイレクト先のURL。
User-Agentブラウザが名乗る自分の名前。
Serverサーバーが身元を名乗るフィールド。
X-Powered-Byサーバーが身元を名乗る非公式フィールド。
:authority, :path, ...HTTP/2以降で使われる疑似ヘッダーフィールド。

コンテンツ情報のフィールド

フィールド名説明
Accept / Content-Type扱える/扱うコンテンツのMIMEタイプ。
Accept-Encoding / Content-Encoding扱える/扱うコンテンツの圧縮形式。
Accept-Language / Content-Language扱える/扱うコンテンツの言語。
Content-Lengthコンテンツのサイズ（バイト）。
Content-Dispositionインライン表示かダウンロードかを指定。
Accept-Ranges / Range / If-Range / Content-Range部分的なダウンロード（Rangeリクエスト）に関連するフィールド群。

キャッシュと通信方式のフィールド

キャッシュ関連

  • Cache-Control: キャッシュ動作を細かく制御。
  • Expires: キャッシュの有効期限（HTTP/1.0）。
  • Last-Modified / If-Modified-Since: 更新日時によるキャッシュ検証。
  • ETag / If-None-Match: コンテンツのハッシュ値などによるキャッシュ検証。
  • Vary: 同じURLでも内容が異なる場合のキャッシュキーを指定。
  • Link: 関連リソースの場所を示す（`rel=preload`など）。
  • Pragma: no-cache: HTTP/1.0互換のキャッシュ無効化ヘッダー。

通信方式関連

  • Connection: `Keep-Alive`, `Close`, `Upgrade`などを指定。
  • Upgrade: アップグレード先のプロトコルを指定。
  • Expect: 100-continue: コンテンツ送信前の確認。
  • Transfer-Encoding: chunked: チャンク転送を示す。
  • Trailer: コンテンツ後に送信するヘッダー名を予告。
  • Via, X-Forwarded-For, Forwarded: プロキシが経由情報を記録。
  • Allow: OPTIONSメソッドへの応答で、利用可能なメソッドを返す。

認証とセキュリティのフィールド

認証・セッション

  • WWW-Authenticate: サーバーが要求する認証方式を通知。
  • Authorization: 認証情報をサーバーに送信。
  • Set-Cookie / Cookie: セッション管理の要。

セキュリティ

  • Content-Security-Policy: リソースの読み込み元を厳格に制限。
  • Strict-Transport-Security: HTTPS接続を強制するHSTS。
  • X-Content-Type-Options: nosniff: MIMEタイプスニッフィングを禁止。
  • Referrer-Policy: Refererヘッダーの送信ポリシーを制御。
  • X-Robots-Tag: 検索エンジンなどのクローラーの動作を制御。

WebSocket関連フィールド

フィールド名説明
Sec-WebSocket-Keyブラウザから送信されるランダムな文字列。
Sec-WebSocket-Acceptサーバーから返される文字列。Keyを元に生成。
Sec-WebSocket-Protocolサブプロトコルを指定する。
Sec-WebSocket-Extensions拡張機能を有効化する。
Sec-WebSocket-VersionWebSocketのバージョン（現在は13で固定）。

CORS関連フィールド

フィールド名説明
Originリクエスト元のオリジンを示す。
Access-Control-Request-Methodプリフライトリクエストで、許可を求めるメソッドを通知。
Access-Control-Request-Headersプリフライトリクエストで、許可を求めるヘッダーを通知。
Access-Control-Allow-Originサーバーがリクエストを許可するオリジンを示す。
Access-Control-Allow-Methodsサーバーが許可するメソッドを示す。
Access-Control-Allow-Headersサーバーが許可するヘッダーを示す。
Access-Control-Allow-Credentialsクッキーの送信を許可するかどうか。
Access-Control-Expose-HeadersJavaScriptからアクセス可能なレスポンスヘッダーを指定。
Access-Control-Max-Ageプリフライトリクエストの結果をキャッシュする期間。

ウェブプッシュ関連フィールド

フィールド名説明
Preferウェブプッシュの設定を指定する。
TTLプッシュ情報の寿命（秒）を設定する。
Topic同じトピックの古いプッシュ情報を上書きするための識別子。
Urgencyメッセージの緊急度を指定（high, normal, low）。

Internet Explorer用の古いフィールド

X-XSS-Protection

IEのXSSフィルターを有効化する。現代のブラウザではCSPに代替された。

X-XSS-Protection: 1; mode=block

X-Frame-Options

クリックジャッキング対策。`frame-ancestors`ディレクティブに代替された。

X-Frame-Options: DENY

X-Content-Security-Policy

IEがサポートするCSP。ディレクティブとしては`sandbox`のみが利用可能。
